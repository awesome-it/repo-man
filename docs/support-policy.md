# Support policy

## Type of support

- **Community-supported open source.** There is no commercial support or SLA.
- **Security updates:** We will address reported security vulnerabilities as described in [SECURITY.md](../SECURITY.md) and release fixes in normal versioned releases (no separate long-term support branch unless stated otherwise).

## End of support period

- **Current support:** Supported for as long as the project is actively maintained (releases and security fixes).
- **End-of-life:** When the project is no longer maintained, we will announce end-of-life in the README and (if applicable) in release notes. After that date we will not provide security updates or new releases.

We do not publish a fixed calendar end-date while the project is actively maintained. If you need a guaranteed support end-date (e.g. for compliance), contact the maintainers (see README or SECURITY.md).

## Versions

Only the latest release line is supported with security fixes. Users are encouraged to upgrade to the latest version.
