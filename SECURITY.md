# Security and vulnerability reporting

## Coordinated vulnerability disclosure

We follow coordinated vulnerability disclosure. If you believe you have found a security vulnerability in repo-man:

1. **Do not** open a public issue for it.
2. **Report it** to the single point of contact below (email preferred).
3. We will acknowledge your report and work with you to understand and address the issue.
4. We aim to fix critical issues promptly and will coordinate with you on disclosure timing (e.g. after a fix is released or after a short embargo).

We do not support disclosure of vulnerabilities through public issue trackers for active security issues.

## SBOM (Software Bill of Materials)

An SBOM can be generated from the project lockfile (e.g. `uv export` or CycloneDX). See [docs/cra-compliance.md](docs/cra-compliance.md) for current status. SBOM is available on request via the contact above until we publish a standard location.
